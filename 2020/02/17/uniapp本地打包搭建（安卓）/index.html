uniapp本地打包搭建（安卓版本）

首先去官网下载sdk（没必要最新的），这里需要注意的是你下载的sdk版本必须与你的HBuildX中cli的编译版本相同，不然可能使用旧版本的HBuildX编译出来的本地资源包可能存在不能增量更新的问题，但是整包更新暂时没有问题。但是如果你使用的编译器版本和sdk版本不一致，必须要在manifest.json文件中的app-plus下面的compatible中添加属性"ignoreVersion" : true

下载好的sdk文件中，找到一个名为HBuilder-Hello的文件夹，切换Android Studio为Project模式，其中app文件夹就是我们后续要执行一系列修改的文件夹，

  • 其中package为应用的包名，采用反向域名格式，为应用的标识，例如： io.uniapp.yzgs
  • versionCode (没做也成功)
    versionCode为应用的版本号（整数值），用于各应用市场的升级判断，建议与manifest.json中version -> code值一致；(如100)
  • versionName (没做也成功)
    versionName为应用的版本名称（字符串），在系统应用管理程序中显示的版本号，建议与manifest.json中version -> name值一致。（如 1.0.0)

解释：

  • 修改main/AndroidMainfest.xml文件中manifest标签的package属性，也就是修改项目的包名，这里要注意的是这里的包名必须要与main/java下的项目包名一致，比如我的项目包名为io.dcloud.HBuilder，那么我需要把这里的package属性也改为io.dcloud.HBuilder，默认一般是相同的。
    • 网上有一种做法是修改了package属性为自己定义的包名，这个包名可以与java文件中的包名字不一致（例如为io.uniapp.yzgs），而且这样也能跑起来，一般没有问题。切换项目文件目录模式为 Android 模式，这时可以看到两个java文件夹，一个为java，另外一个为java(generated)，java文件夹就是project文件模式下的java文件夹，下面存在包io.docloud.HBuild,而java(generated)文件夹是根据main/AndroidMainfest.xml生成出来的，里面存在一个名为io.uniapp.yzgs的包。

main/AndroidMainfest.xml文件中manifest标签的package属性值与项目的包名不一致会出现什么问题？

  • 问题的背景： 我的项目的更新逻辑需要开启自动安装权限，但是安卓8以上的系统，需要在配置REQUEST_INSTALL_PACKAGES权限，网上有两种做法，第一种是在uniapp项目在打包之前，在mainfest.json文件中添加"<uses-permission android:name=\"android.permission.REQUEST_INSTALL_PACKAGES\"/>"这样一个配置项，具体的添加位置在distribute.android.permissions下面，第二种做法是在安卓项目中的main/AndroidMainfest.xml文件中添加<uses-permission android:name="android.permission.REQUEST_INSTALL_PACKAGES" />，经过测试，第一种方法适用于云打包的项目，对本地打包的项目完全没有作用，而且打包出来的__UNI__A011DA1/www/manifest.json整合了uniapp项目manifest.json文件和pages.json中的部分信息，而且__UNI__A011DA1/www/manifest.json中并没有包含"<uses-permission android:name=\"android.permission.REQUEST_INSTALL_PACKAGES\"/>"这种配置项信息。第二种方法更加适用于本地打包的项目，因为原生的安卓项目就是这样配置权限的。
    • 解决过程： 但是我在项目种使用了第二种做法，app可以正常运行，但是在更新的时候依然没能调用系统的更新功能，具体的表现就是下载安卓包直到100%都没有问题，但是执行plus.runtime.install的时候就会直接执行回调函数中的退出应用方法（因为我这里的逻辑是调出系统安装应用界面，并且关闭当前应用），甚至没有出现网上流传的安卓8以上系统因为没有配置请求安装权限而引起的安装界面闪过现象。
    • 找到原因： 其实我的权限配置是没有问题的，权限不生效的原因是main/AndroidMainfest.xml文件中的package属性名与我的项目包名不一致。经过查阅发现，安卓8在配置了这个权限之后，还需要应用的activity文件种编写一段代码，我猜测这段代码应该写在了arr包中，但是我如果写的配置文件中的包名与项目包名不一致的话，最终执行的生成的包，即使配置了权限，但是没有配置权限后还要执行的代码，这样就导致了权限配置了，但对应用不生效。
